## 設備認證與 Token 管理

### 設備標識

- 設備可以自動獲取設備碼，用於唯一標記當前設備
- 設備碼用於後續所有接口請求中的設備身份識別

### 設備綁定

- 通過設備碼接口請求當前設備所綁定的資訊：
  - 大廈信息
  - 設備配置信息
  - 認證 Token

### Token 管理

- **Token 存儲**：Token 需要持久化存儲在應用數據中
- **Token 使用**：所有數據獲取接口（廣告、通告、天氣等）都需要使用 Token 進行身份認證
- **Token 刷新**：當發現 Token 過期時，應當自動通過認證接口重新獲取 Token
- **認證流程**：
  1. 檢測到 Token 過期（API 返回 401 或特定錯誤碼）
  2. 自動調用認證接口獲取新 Token
  3. 更新本地存儲的 Token
  4. 重新執行原來的 API 請求

## 配置參數

以下參數需要持久化存儲在應用數據中，用戶可配置：

| 參數名稱         | 描述                                               | 默認值 |
| ---------------- | -------------------------------------------------- | ------ |
| 閒置時間         | 手動操作狀態下，多久沒有用戶操作後自動轉為輪播狀態 | 30 秒  |
| 輪播持續時間     | 自動輪播狀態持續多久後進入全屏廣告狀態             | 300 秒 |
| 全屏廣告持續時間 | 全屏廣告狀態持續多久後返回自動輪播狀態             | 60 秒  |
| 通告輪播時長     | 輪播狀態下每個通告的顯示時長                       | 10 秒  |
| 天氣切換間隔     | 天氣/資訊內容的切換間隔                            | 15 秒  |

## 用戶操作定義

用戶操作包括所有觸摸屏幕的行為：

- 點擊（單擊、雙擊）
- 滑動（上下左右滑動）
- 拖拽
- 縮放手勢
- 其他多點觸控操作

## 狀態切換

手動操作狀態下，檢測到閒置時間內沒有用戶操作，轉移到自動輪播狀態
自動輪播狀態下，監測到有用戶操作，轉移到手動操作狀態
自動輪播狀態下，經過輪播持續時間並且全屏廣告隊列不為空，轉移到全屏廣告狀態
全屏廣告狀態下，監測到用戶操作，轉移到手動操作狀態並且退回主屏幕
全屏廣告狀態下，經過全屏廣告持續時間，轉移到自動輪播狀態

## 全屏廣告狀態

### 廣告隊列

需要維護一個全屏廣告循環隊列，用來存儲一系列要播放的全屏廣告，並且記錄當前播放到哪一條廣告、當前廣告播放了多久。
每一個廣告有：一個廣告文件（視頻或圖片）、本條廣告播放時長
當前廣告播放時長達到本條廣告播放時長時，切換到廣告隊列中的下一個廣告

支持的廣告文件格式：

- 視頻：mp4
- 圖片：jpg, jpeg, png, gif

### 狀態切換

全屏廣告狀態維持時間由用戶設定，用來決定何時進行狀態轉換。每次切換到全屏廣告狀態時，都要有一個從 0 開始的計時器用來記錄在全屏廣告狀態維持了多久，當全屏廣告計時器時長達到用戶設定的全屏廣告狀態時長時，需要切換到自動輪播狀態。當檢測到用戶點擊屏幕任何位置時，要切換到手動操作狀態。
在切換狀態時，需要保留廣告隊列的數據（存儲在內存中），以便下一次進入全屏廣告狀態時，可以繼續播放廣告隊列中的廣告。

## 自動輪播狀態

自動輪播狀態下，頁面從上到下分為 3 部分：頂部廣告輪播、通告輪播、天氣/資訊輪播
自動輪播狀態下需要輪播：PDF 類型的通知、圖片類型的通知、自定義的頁面

### 頂部廣告輪播

需要維護一個頂部廣告循環隊列，用來存儲一系列要播放的頂部廣告，並且記錄當前播放到哪一條廣告、當前廣告播放了多久。
每一個廣告有：一個廣告文件（視頻或圖片）、本條廣告要播放的時長
當前廣告播放時長達到本條廣告播放時長時，切換到廣告隊列中的下一個廣告

支持的廣告文件格式：

- 視頻：mp4
- 圖片：jpg, jpeg, png, gif

### 通告輪播

需要維護一個輪播循環隊列，用於記錄當前輪播到哪裡。
隊列中的元素有：輪播類型（PDF/圖片/自定義頁面）、輪播文件（PDF/圖片類型下）、輪播數據（自定義頁面類型下）

PDF 顯示方式：支持上下拖動瀏覽形式

### 天氣/資訊

通過 HTTP 接口獲取天氣/資訊，每間隔天氣切換間隔時間切換到下一條

### 狀態切換

自動輪播狀態下，用戶手動觸摸屏幕可以轉換到手動操作模式，模式切換時，需要保留當前自動輪播的現場（通告播放到哪裡了）
自動輪播狀態下，需要維護一個計時器用來記錄已經自動輪播了多久，當輪播時長達到用戶設定的輪播持續時間時，切換到全屏廣告狀態，並且保留現場（通告、廣告、天氣/資訊分別播放到哪裡了），以備返回時用

## 手動操作狀態

手動操作狀態下，頁面與自動輪播狀態佈局相同，並且頂部廣告、天氣輪播正常進行，只有中間的通告輪播會暫停
手動操作狀態下，需要維護一個計時器，用來記錄有多久沒有用戶操作。當用戶操作頁面時應當刷新該計時器。
當計時器到達閒置時間設定值時，自動切換到自動輪播狀態。

## 數據結構

### 全屏廣告隊列數據結構

```
{
  currentIndex: number,        // 當前播放廣告索引
  currentPlayTime: number,     // 當前廣告已播放時間
  ads: [
    {
      id: string,              // 廣告ID
      file: string,            // 廣告文件路徑/URL
      duration: number,        // 播放時長（秒）
      type: 'video' | 'image'  // 廣告類型
    }
  ]
}
```

### 頂部廣告隊列數據結構

```
{
  currentIndex: number,        // 當前播放廣告索引
  currentPlayTime: number,     // 當前廣告已播放時間
  ads: [
    {
      id: string,              // 廣告ID
      file: string,            // 廣告文件路徑/URL
      duration: number,        // 播放時長（秒）
      type: 'video' | 'image'  // 廣告類型
    }
  ]
}
```

### 通告隊列數據結構

```
{
  currentIndex: number,        // 當前播放通告索引
  announcements: [
    {
      id: string,              // 通告ID
      type: 'pdf' | 'image' | 'custom',  // 通告類型
      file?: string,           // 文件路徑/URL（PDF/圖片類型）
      data?: any              // 自定義數據（自定義頁面類型）
    }
  ]
}
```

## 錯誤處理

### 廣告文件加載失敗

- 跳過失敗的廣告，繼續播放隊列中的下一個廣告
- 記錄錯誤日誌

### 網絡異常

- 天氣/資訊獲取失敗時，顯示網絡異常提示
- 廣告/通告數據獲取失敗時，顯示相應錯誤提示

### Token 認證失敗

- 當接口返回 401 未授權錯誤時，自動執行 Token 刷新流程
- 如果 Token 刷新失敗，顯示認證錯誤提示
- 記錄認證錯誤日誌

### 設備綁定失敗

- 當設備碼無法獲取設備綁定信息時，顯示設備未綁定提示
- 提供重新嘗試或聯繫管理員的選項

### 隊列為空

- 全屏廣告隊列為空時，播放默認圖片
- 頂部廣告隊列為空時，播放默認圖片
- 通告隊列為空時，顯示默認通告內容

## 應用初始化

應用啟動時執行以下初始化流程：

1. **設備認證初始化**

   - 獲取設備碼
   - 通過設備碼接口獲取設備綁定信息（大廈、配置、Token）
   - 將 Token 存儲到應用數據中

2. **加載配置參數**

   - 從應用數據中讀取用戶配置的時間參數
   - 從服務器獲取的設備配置信息覆蓋本地配置
   - 如無配置則使用默認值

3. **獲取廣告數據**

   - 通過 HTTP 接口獲取廣告列表（使用 Token 認證）
   - 根據廣告類型分別加載到全屏廣告隊列和頂部廣告隊列

4. **獲取通告數據**

   - 通過 HTTP 接口獲取通告列表（使用 Token 認證）
   - 將通告數據加載到通告隊列

5. **獲取天氣資訊**

   - 通過 HTTP 接口獲取天氣/資訊數據（使用 Token 認證）

6. **初始狀態**
   - 應用啟動後默認進入自動輪播狀態

## 數據存儲說明

- **配置參數**：持久化存儲在應用數據中
- **認證 Token**：持久化存儲在應用數據中
- **設備綁定信息**：持久化存儲在應用數據中
- **廣告隊列、通告隊列、狀態數據**：存儲在內存中，應用關閉後消失
- **現場保留數據**：臨時存儲在內存中，用於狀態切換時的數據恢復

## HTTP 接口規範

### 設備綁定接口

```
GET /api/device/binding?deviceCode={設備碼}
Response: {
  buildingInfo: {
    id: string,
    name: string,
    // 其他大廈信息
  },
  deviceConfig: {
    // 設備配置參數
  },
  token: string
}
```

### 認證接口

```
POST /api/auth/refresh
Body: {
  deviceCode: string
}
Response: {
  token: string,
  expiresIn: number
}
```

### 廣告列表接口

```
GET /api/ads
Headers: {
  Authorization: Bearer {token}
}
Response: {
  fullScreenAds: [...],
  topAds: [...]
}
```

### 通告列表接口

```
GET /api/announcements
Headers: {
  Authorization: Bearer {token}
}
Response: {
  announcements: [...]
}
```

### 天氣資訊接口

```
GET /api/weather
Headers: {
  Authorization: Bearer {token}
}
Response: {
  weather: [...]
}
```
