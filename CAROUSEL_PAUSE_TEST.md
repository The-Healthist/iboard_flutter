# 轮播暂停/恢复功能测试指南

## 功能概述
本功能实现了"进入到全屏广告状态时，暂停通告轮播、顶部广告轮播"的需求。

## 已实现的功能

### 1. 状态管理
- **CarouselStateProvider**: 管理应用状态（默认、全屏广告、手动操作）
- **暂停状态**: `_isCarouselPaused` 布尔变量控制所有轮播的暂停
- **状态监听**: 在 `build()` 方法中监听状态变化

### 2. 轮播控制
- **顶部广告轮播**: 动态时长，支持暂停/恢复
- **中部通告轮播**: 15秒间隔，支持暂停/恢复  
- **底部组件轮播**: 10秒间隔，支持暂停/恢复

### 3. 暂停/恢复逻辑
- **暂停**: `_pauseAllCarousels()` 方法取消所有定时器
- **恢复**: `_resumeAllCarousels()` 方法重新启动所有定时器
- **状态检查**: 所有定时器在执行前检查 `!_isCarouselPaused`

## 测试步骤

### 准备工作
1. 确保应用已加载广告数据（顶部和全屏）
2. 确保有多个通告数据用于轮播测试
3. 观察各轮播区域是否正常自动切换

### 测试场景 1: 进入全屏广告
1. **预期行为**: 
   - 点击全屏广告或自动进入全屏广告状态
   - 所有轮播应立即暂停
   - 控制台输出: `🛑 暂停所有轮播 - 进入全屏广告状态`

2. **验证要点**:
   - 顶部广告轮播停止
   - 中部通告轮播停止
   - 底部组件轮播停止
   - 当前显示内容保持不变

### 测试场景 2: 退出全屏广告
1. **预期行为**:
   - 从全屏广告返回到主界面
   - 所有轮播应立即恢复
   - 控制台输出: `▶️ 恢复所有轮播 - 退出全屏广告状态`

2. **验证要点**:
   - 顶部广告轮播恢复（按照广告自身时长）
   - 中部通告轮播恢复（15秒间隔）
   - 底部组件轮播恢复（10秒间隔）
   - 轮播从当前位置继续，不重置

### 测试场景 3: 多次切换
1. **测试步骤**:
   - 进入全屏广告 → 退出 → 再次进入 → 再次退出
   - 重复多次测试状态切换的稳定性

2. **验证要点**:
   - 每次暂停都能正确停止所有轮播
   - 每次恢复都能正确重启所有轮播
   - 没有内存泄漏或定时器残留
   - 状态切换响应及时

## 技术实现细节

### 关键代码位置
- **状态提供者**: `lib/providers/state_provider.dart`
- **主要逻辑**: `lib/pages/mainscreen_page.dart`
- **暂停方法**: `_pauseAllCarousels()`
- **恢复方法**: `_resumeAllCarousels()`
- **状态监听**: `build()` 方法中的 `context.watch<CarouselStateProvider>()`

### 定时器保护机制
```dart
// 所有定时器都有双重保护
if (mounted && _midCarouselController.widgetCount > 1 && !_isCarouselPaused) {
  _midCarouselController.playNext();
}
```

### 状态切换日志
- 进入全屏: `🛑 暂停所有轮播 - 进入全屏广告状态`
- 退出全屏: `▶️ 恢复所有轮播 - 退出全屏广告状态`

## 可能的扩展功能

### 1. 精确状态恢复
- 保存暂停时的精确位置和剩余时间
- 恢复时从保存的状态继续，而不是重新开始

### 2. 视频播放控制
- 如果广告包含视频，暂停时停止视频播放
- 恢复时重新开始视频播放

### 3. 动画暂停
- 暂停轮播切换动画
- 暂停广告内部的动画效果

## 已知限制

1. **简单恢复**: 当前实现是简单的暂停/恢复，不保存精确的暂停位置
2. **视频处理**: 尚未实现视频广告的暂停控制
3. **动画控制**: 尚未实现广告内动画的暂停
